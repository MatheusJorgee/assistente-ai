### Problema Resolvido
`sqlite3.OperationalError: no such table: messages` após apagar ficheiro .db

### Solução Implementada

#### 1. Método `_init_db()` Automático
```python
def __init__(self, db_path: Optional[str] = None):
    # ...
    self._init_db()  # ✅ Chamado no __init__
```

#### 2. Três Tabelas Criadas com CREATE TABLE IF NOT EXISTS
- **messages**: id, session_id, role, content, timestamp, metadata
- **events**: id, session_id, event_type, data, timestamp
- **images**: id, session_id, base64_data, width, height, format, timestamp

#### 3. Tratamento de Erros Robusto
- try/except/finally em `_save_message_to_db()`
- try/except/finally em `_save_event_to_db()`
- try/except/finally em `_save_image_to_db()`
- try/except/finally em `_clear_session_from_db()`

#### 4. Segurança e Performance
- Thread-safe com `threading.RLock()`
- Logging detalhado (debug/info/error)
- Memória em RAM + Persistência em SQLite
